# VS Protocol Web Interface

虚拟筛选流程的图形化管理界面

## 功能特性

### 🎯 核心功能

1. **配置管理**
   - 加载和编辑YAML配置文件
   - 实时保存配置更改
   - 模块快捷开关（一键启用/禁用各个模块）
   - 配置重置功能

2. **流程控制**
   - 一键启动虚拟筛选流程
   - 实时停止正在运行的流程
   - 流程状态监控

3. **进度监控**
   - 实时显示当前运行的模块
   - 进度条显示整体完成度
   - 可视化流程图展示各模块状态
   - 自动高亮当前运行模块

4. **日志系统**
   - 实时显示运行日志
   - 日志分级显示（info/success/warning/error）
   - 日志自动滚动
   - 日志导出功能

## 使用指南

### 配置管理

1. **选择配置文件**
   - 在左侧面板的下拉菜单中选择配置文件（如 `config_loose.yaml`）
   - 配置内容会自动加载到编辑器中

2. **编辑配置**
   - 直接在文本框中编辑YAML配置
   - 或使用下方的模块快捷开关快速启用/禁用模块

3. **保存配置**
   - 点击"💾 保存配置"按钮保存更改
   - 点击"↩️ 重置"按钮恢复到上次保存的状态

### 启动流程

1. 确保已选择并配置好配置文件
2. 点击右侧的"▶️ 启动流程"按钮
3. 确认对话框后，流程将在后台开始运行

### 监控进度

- **状态信息**：显示当前运行状态、模块和开始时间
- **进度条**：显示整体完成百分比
- **流程图**：可视化展示各模块的运行状态
  - 灰色：未开始
  - 绿色：已完成
  - 蓝色（闪烁）：正在运行

### 查看日志

- 日志会实时显示在右下方的日志面板中
- 不同级别的日志用不同颜色标识：
  - 白色：普通信息
  - 绿色：成功信息
  - 黄色：警告信息
  - 红色：错误信息
- 可以清空日志或导出为文本文件

## 技术架构

### 后端

- **Flask**：Web框架
- **Flask-SocketIO**：WebSocket实时通信
- **PyYAML**：YAML配置文件处理
- **subprocess**：运行虚拟筛选流程

### 前端

- **原生JavaScript**：无需额外框架
- **Socket.IO**：实时数据推送
- **现代CSS**：响应式设计

### 通信流程

```
浏览器 <--WebSocket--> Flask服务器 <--subprocess--> vs_protocol.py
```

## 常见问题

### 1. 无法访问Web界面

- 检查防火墙是否开放5000端口
- 确认Flask服务器正在运行
- 尝试使用 `0.0.0.0:5000` 而不是 `localhost:5000`

### 2. 流程启动后没有反应

- 检查配置文件路径是否正确
- 查看日志面板中的错误信息
- 确认 `vs_protocol.py` 可以正常运行

### 3. 进度不更新

- 刷新浏览器页面
- 检查WebSocket连接状态（F12开发者工具 -> Console）
- 重启Flask服务器

### 4. 配置保存失败

- 检查YAML格式是否正确
- 确认文件权限允许写入
- 查看浏览器控制台的错误信息

## 安全建议

⚠️ **重要提示**：

1. **不要在公网直接暴露**：此界面未包含身份验证，仅适合内网使用
2. **使用反向代理**：如需公网访问，建议使用Nginx配置HTTPS和基本认证
3. **定期备份配置**：重要配置文件应定期备份
4. **限制访问权限**：确保只有授权用户可以访问服务器
